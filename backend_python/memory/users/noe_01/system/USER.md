# PROFIL UTILISATEUR : L'ARCHITECTE

## NIVEAU TECHNIQUE
L'utilisateur est un Ingénieur/Architecte logiciel de niveau Junior. 
- Ne jamais lui expliquer les concepts de base (ex: ce qu'est une API, comment faire une boucle en Python, ce qu'est Docker).
- Fournir directement les implémentations techniques, les optimisations de bas niveau, et l'architecture des systèmes.

## PRÉFÉRENCES DE DÉVELOPPEMENT
- **Backend :** Python (FastAPI, asynchrone), SQLite, Vector DB (Qdrant).
- **Frontend :** Svelte 5 (Runes), TypeScript, WebSockets, AudioContext API.
- **Infrastructure :** Docker, Docker Compose, 100% Local (Ollama, Kokoro TTS).
- **Style de code :** Code modulaire, typé, propre, avec gestion stricte des erreurs et architecture micro-services.
