# 🎩 Jean-Heude

**J.E.A.N-H.E.U.D.E** is an advanced AI orchestrator designed to be **100% local, multimodal, multi-channel, and fully open-source**.

By leveraging cutting-edge community projects, it provides a sovereign AI experience without compromising on privacy, security, or performance. Built on a modernized **Clean Architecture**, Jean-Heude can hear, see, and "think" across numerous platforms and databases in real-time.

---

## 🚀 Key Features

### 🛡️ Total Sovereignty & Zero-Trust Security
100% local execution—your private data, files, and images never leave your machine. Interactions are securely isolated per user via **JSON Web Tokens (JWT)** over FastApi, securing both HTTP requests and bidirectional WebSockets. A built-in SQLite allowlist ensures the bot refuses interaction until securely authenticated via a `/pair` command on external networks.

### 🧠 Clean Architecture & Paralleled Context
The core reasoning loop runs on a highly scalable, non-blocking architecture:
- **Universal Memory Manager:** A centralized facade delegating context retrieval asynchronously.
- **Hybrid RAM + DB Retrieval:** Context generation happens simultaneously via `asyncio.gather()`, aggregating instant session memory, dense vector knowledge (Qdrant), and entity ontology (Neo4j) in milliseconds before the LLM even begins to think.

### 👁️ Multimodal Vision & Dynamic Tools
Powered by the **Model Context Protocol (MCP)**, the AI doesn't just chat; it autonomously selects and executes specific **Skills** to fetch real-time news, extract OSINT data, or interact with your file system. It can also "see" and analyze visual documents directly within the chat via multimodal vision models (e.g., LLaVA, Moondream).

### 🎙️ Ultra-Smooth Voice Interaction (PCM Bitstreaming)
Unlike traditional systems that generate a full **WAV** file (requiring the user to wait for the end of generation), Jean-Heude utilizes **raw 16-bit PCM bitstreaming**:
- **Zero Wait Latency:** Audio starts playing the moment the first bytes are generated by the TTS engine.
- **Intelligent Sequencing:** A modern Frontend **Promise Chain** queues audio IDs. Sentences flow naturally without gaps, overlaps, or robotic pauses.
- **Bit-Perfect Stability:** Advanced handling of leftover bytes ensures 16-bit samples split across packets are reconstructed flawlessly.

### 🌐 Multi-Channel & Component-Driven UI
Interact seamlessly via the modernized **SvelteKit Web UI**, **Telegram**, or **Discord**. The web dashboard is fully scalable, built upon distinct reactive components (Sidebar, ChatContainer, Player) leveraging the new Svelte 5 Runes mechanism.

## 🛠 Technical Stack

The architecture is split into specialized micro-services for maximum efficiency:

- **Frontend:** **SvelteKit** (Svelte 5) utilizing the **AudioContext API** for real-time binary stream rendering.
- **Backend Gateways:** **Python (FastAPI)** handling JWT web-sockets, alongside asynchronous polling for **Discord (`discord.py`)** and **Telegram (`python-telegram-bot`)**.
- **Memory Layer (MemoryManager):** 
  - **Qdrant**: Long-term semantic/vector retrieval.
  - **Neo4j**: GraphRAG database for real-time ontology and relationship tracking.
  - **SQLite**: Instant session tracking, user identity management, and keyword indexing.
- **Inference Engine:** **Ollama**, serving Large Language Models (LLMs) and Embeddings.
- **Voice Engine:** **Kokoro-ONNX** for high-quality, local Text-to-Speech (TTS).
- **Orchestration:** Fully containerized via **Docker Compose** for a seamless, production-ready deployment.

---

## 📦 Infrastructure Requirements

To get Jean-Heude up and running, you will need:

1. **Ollama:** Running on your host machine to execute models.
2. **Docker & Docker Compose:** To orchestrate the stack (`docker-compose.yml` included).
3. **Voice Models:** `kokoro-v1.0.onnx` and `voices-v1.0.bin` files placed in the `backend_python/tts/` directory.
4. **Environment Variables:** A `.env` file containing variables like `TELEGRAM_BOT_TOKEN`, `DISCORD_BOT_TOKEN`, and `URL_SERVER_OLLAMA`.

## 📈 Version History

### **V5.1 (Current) — The "MCP Expansion" & Optimization Era**
- **worldmonitor-mcp integration:** 107 tools across 32 services (markets, military, geopolitics, DeFi, SEC EDGAR, CFTC, conflict data...) replacing the previous world monitor. Configured via `WORLDMONITOR_BASE_URL`.
- **Tool embeddings cache:** SHA256-based invalidation — embeddings are computed once and stored in a local JSON file. Subsequent restarts load from cache (~20s saved). Auto-invalidated on `mcp_servers.yaml` or skill manifest changes.
- **JIT tool selection tuning:** Cosine similarity threshold raised to 0.62 (was 0.5) and limits tightened (10 for model selection, 12 for inference) — LLM receives fewer, more relevant tools.
- **Multimodal pipeline refactor:** Vision model (`minicpm-v:8b`) now runs without tools (function calling is unstable on small vision models). Image analysis is injected as context into `process_chat` which handles reasoning and tool use.
- **UI fixes:** User chat bubbles right margin, sidebar gear button alignment (`flex-shrink: 0`, `box-sizing: border-box`).
- **Docker build speed:** `src-tauri` (9.4GB Rust artifacts) excluded from frontend Docker context — build time reduced from ~600s to ~30s.
- **URL injection via Docker ARG:** `PUBLIC_URL_SERVEUR_PYTHON` baked at build time via `.env.local`, not at runtime. Eliminates `$env/dynamic/public` SSR crash.

### **V5.0 — The "Clean Architecture" & Security Era**
- **Robust Security overhaul:** Complete replacement of raw websocket payloads with a strict **JWT (JSON Web Token)** authentication middleware across all FastAPI routes.
- **Memory Manager Facade:** Eradication of "God Classes" (`AgentRunner` and `MemoryIA`). Database interactions are firmly decoupled via a unified backend Manager.
- **Massive Performance Boost:** Context gathering for the System Prompt operates in full asynchronous parallel (Neo4j + Qdrant + WebCache + SQLite) saving roughly 60% of idle reasoning latency.
- **Svelte 5 Frontend Refactor:** Total decomposition of the massive monothilic page into scalable, bite-sized components (`Sidebar`, `ChatInput`).

### **V4.6 — The Omni-Channel & Brain Era**
- **Vision Integration:** The AI now "sees" and processes images natively.
- **External Gateways:** Native background integrations for Discord and Telegram.
- **Micro-services Architecture:** First formal Containerization of the stack via Docker Compose.

### **V4.5 — The Voice Era**
- **Sequential TTS Engine:** Breakthrough transition from WAV files to **24kHz PCM streaming** via Proxy.
- **Integrated STT:** High-accuracy Speech-to-Text for hands-free interaction.

### **V4.0 — Agentic Loop**
- **MCP Support:** Integration of the Model Context Protocol for dynamically pulling external tools.
- **Visual Thinking:** Distinct UI blocks that display the hidden "Thinking" chain in real-time.
